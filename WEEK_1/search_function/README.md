# Search Function

## Objective
Implement Linear Search and Binary Search for an e-commerce platform.

## Big O Notation
Big O notation measures how the running time of an algorithm grows with the size of the input.

## Time Complexity

### Linear Search
- Best Case: O(1)
- Average Case: O(n)
- Worst Case: O(n)

### Binary Search
- Best Case: O(1)
- Average Case: O(log n)
- Worst Case: O(log n)

## Comparison

| Algorithm | Best | Average | Worst |
|-----------|------|---------|-------|
| Linear Search | O(1) | O(n) | O(n) |
| Binary Search | O(1) | O(log n) | O(log n) |

## Conclusion
Binary Search is more efficient for large sorted datasets because it reduces the search space by half in each iteration.
