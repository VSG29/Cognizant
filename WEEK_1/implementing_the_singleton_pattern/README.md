# Singleton Pattern

## Objective
Implement the Singleton Design Pattern to ensure that only one instance of the Logger class is created throughout the application.

## Classes
- Logger.java
- TestLogger.java

## Design Pattern Used
Singleton Pattern

## Advantages
- Ensures only one object is created.
- Saves memory.
- Provides a global access point.
- Useful for logging, configuration, and database connections.

Logger Instance Created
Log: First message
Log: Second message
